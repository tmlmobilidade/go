Pipeline reference

1. SNAP — every raw GPS ping is matched to the nearest node of the matching shape using a geohash-7 + bbox + argMin spatial join. Events farther than max_dist_m from any node are discarded.

2. FORWARD-ONLY FILTER — per ride (sorted by timestamp), drop events whose snapped node index goes backwards relative to the previous event (uses lag(), not running max(), so a single outlier cannot poison a whole ride).

3. PAIR & VALIDATE — pair each event with its predecessor as a segment. A segment is kept only if all of:

  • curr_idx > prev_idx (must advance)
  • curr_ts − prev_ts > 0
  • implied speed ∈ [1, 120] km/h
  • event bearing close to node bearing (|Δθ| < 90°)

4. EXPAND — each valid segment becomes one row per traversed node, distributing seconds evenly (base_sec + leading rem_sec nodes get +1s).

5. RE-TIME — per-node travel time = gap to the next emitted node timestamp on the (ride, shape) timeline using lead(). Last node of each ride/shape gets 0.
